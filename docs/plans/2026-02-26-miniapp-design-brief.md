# ТЗ для дизайнера: Вики Diary — Telegram Mini App

**Дата:** 26 февраля 2026
**Проект:** Вики Diary
**Платформа:** Telegram Mini App (WebApp)
**Формат доставки:** Figma-макеты + кликабельный прототип

---

## 1. О проекте

**Вики Diary** — детский дневник внутри Telegram. Родители отправляют боту фото, видео, текст и голосовые сообщения — бот сохраняет всё в хронологическую ленту, автоматически расставляет теги (AI) и генерирует ежемесячные саммари.

Сейчас взаимодействие — только через чат-бота. Mini App добавляет полноценный визуальный интерфейс: удобный просмотр, создание и редактирование записей, саммари и статистика.

**Бот остаётся основным каналом** для быстрого добавления записей (фото из чата, голосовые). Mini App — для просмотра, редактирования и аналитики.

---

## 2. Целевая аудитория

**Молодые родители 25-35 лет** — мамы и папы младенцев и тоддлеров.

Характеристики:
- Технически грамотные, привыкли к Instagram/Pinterest-эстетике
- Пользуются Telegram ежедневно
- Ценят скорость — записи делаются «на ходу», одной рукой
- Эмоционально вовлечены — дневник ребёнка = ценный семейный артефакт

---

## 3. Задачи дизайнера

1. Разработать логотип и цветовую палитру
2. Спроектировать UI Kit с базовыми компонентами
3. Отрисовать все экраны со всеми состояниями
4. Собрать кликабельный прототип основных сценариев
5. Создать иллюстрации для пустых состояний

---

## 4. Карта экранов и навигация

```
┌─────────────────────────────────────────────┐
│                 НАВИГАЦИЯ                    │
│  [Лента]    [Создать +]    [Саммари]        │
│  (bottom tab bar)                           │
└─────────────────────────────────────────────┘

1. ЛЕНТА ЗАПИСЕЙ (главный экран)
   └── 2. ДЕТАЛЬ ЗАПИСИ (полный просмотр)
       └── 3. РЕДАКТИРОВАНИЕ ЗАПИСИ (модальное)

4. СОЗДАНИЕ ЗАПИСИ

5. САММАРИ И СТАТИСТИКА
   ├── Месячное саммари (AI-текст)
   └── Статистика по тегам (визуализация)
```

**Навигация:**
- Bottom tab bar с 3 табами: **Лента** / **Создать (+)** / **Саммари**
- Переход в деталь записи — push-навигация (с кнопкой «Назад»)
- Редактирование — модальное окно поверх детали
- Telegram Mini App header используется для заголовка + нативной кнопки «Назад»

## 5. Описание экранов

### 5.1. Лента записей (главный экран)

**Назначение:** Хронологическая лента всех записей дневника.

**Элементы:**
- **Заголовок:** имя ребёнка + возраст (пример: «Вика — 1 год 3 мес»)
- **Список записей**, сгруппированных по дням
  - Разделитель — дата в длинном формате: «26 февраля 2026»
- **Карточка записи:**
  - Аватарка автора + имя (Серёжа / Настя) + время «14:30»
  - Превью текста (2-3 строки, обрезка с троеточием)
  - Миниатюры фото/видео — сетка 2×2; если > 4 — бейдж «+3»
  - Теги-чипсы внизу карточки (#первый-раз #еда #прогулка)
- **Pull-to-refresh** для обновления
- **Infinite scroll** (подгрузка при скролле вниз, от новых к старым)

**Состояния:**
| Состояние | Описание |
|---|---|
| Загрузка | Skeleton-шиммер (3-4 карточки-заглушки) |
| Пустой дневник | Иллюстрация + текст «Начните вести дневник — отправьте фото или сообщение боту» |
| Ошибка сети | Иллюстрация + кнопка «Повторить» |
| Заполненный | Нормальный вид с карточками |

---

### 5.2. Деталь записи

**Назначение:** Полный просмотр одной записи со всеми медиа.

**Элементы:**
- **Шапка:** аватарка автора + имя + дата события + время создания
- **Полный текст** записи (без обрезки)
- **Медиа-галерея:**
  - Фото: полноширинные; по тапу — полноэкранный просмотр с зумом и свайпом между фото
  - Видео: превью-кадр с кнопкой Play; воспроизведение инлайн
  - Голосовые: мини-плеер с визуализацией волны и длительностью
- **Теги** внизу контента
- **Кнопки действий:** «Редактировать» / «Удалить»

**Состояния:**
| Состояние | Описание |
|---|---|
| Загрузка медиа | Placeholder (blur или skeleton) |
| Подтверждение удаления | Bottom sheet: «Удалить запись? Это действие нельзя отменить» — кнопки «Удалить» (красная) и «Отмена» |
| Полноэкранный просмотр фото | Overlay с зумом, свайпом, кнопкой закрытия |

---

### 5.3. Создание записи

**Назначение:** Добавление новой записи из Mini App.

**Элементы:**
- **Поле ввода текста** — multiline, автовысота, placeholder «Что сегодня произошло?»
- **Кнопка прикрепления** фото/видео (из галереи устройства)
- **Превью прикреплённых медиа** — горизонтальная лента с крестиками для удаления каждого файла
- **Выбор даты события:**
  - По умолчанию «Сегодня»
  - Быстрые опции: «Вчера», «Позавчера»
  - Ссылка «Другая дата» → календарь
- **Кнопка «Сохранить»** — нативный Telegram MainButton внизу экрана

**Состояния:**
| Состояние | Описание |
|---|---|
| Пустая форма | Placeholder в поле, дата = Сегодня |
| Заполненная форма | Текст + прикреплённые медиа |
| Отправка | Лоадер на MainButton, форма неактивна |
| Успех | Снэкбар «Записано!» → автовозврат в ленту |

---

### 5.4. Редактирование записи

**Назначение:** Изменение текста и даты существующей записи.

**Элементы:**
- Аналогично экрану «Создание», но с предзаполненными данными
- **Текст** — редактируемый, предзаполнен текущим текстом
- **Медиа** — отображаются как read-only (без удаления/добавления) — ограничение MVP
- **Дата события** — изменяемая
- **Кнопка «Сохранить изменения»** — Telegram MainButton

**Состояния:**
| Состояние | Описание |
|---|---|
| Загрузка данных | Skeleton в полях |
| Редактирование | Предзаполненная форма |
| Сохранение | Лоадер на MainButton |
| Успех | Возврат к детали записи |

---

### 5.5. Саммари и статистика

**Назначение:** AI-сгенерированное месячное саммари + визуальная статистика.

**Элементы:**
- **Переключатель месяцев:** ← Январь 2026 →
- **Блок «Саммари»:**
  - AI-текст (300-800 слов)
  - Оформление — как уютная «страница из дневника» или «открытка»
  - Тёплая типографика, может быть слегка декоративный фон
- **Блок «Статистика»:**
  - Количество записей за месяц (крупная цифра)
  - Топ тегов — горизонтальные барчарты или облако тегов
  - Календарь-хитмап: дни с записями подсвечены интенсивностью (как на GitHub, но тёплыми цветами)

**Состояния:**
| Состояние | Описание |
|---|---|
| Генерация саммари | Анимация ожидания (~5-10 сек), приятная, не агрессивная |
| Нет записей за месяц | Иллюстрация + «В этом месяце записей пока нет» |
| Будущий месяц | Кнопка «→» неактивна |
| Готово | Саммари + статистика отображены |

## 6. Визуальный стиль

### Настроение

Тёплый, уютный, как бумажный детский дневник в цифровом формате. Никакого «корпоративного» или «технического» ощущения. Приложение должно вызывать улыбку.

### Цветовая палитра (направление)

| Роль | Направление |
|---|---|
| Основной (primary) | Мягкий коралловый или персиковый |
| Фон | Тёплый белый / кремовый |
| Акценты | Приглушённый розовый, мятный, лавандовый |
| Текст | Тёмно-серый (#333-#444), не чёрный |
| Успех | Смягчённый зелёный |
| Ошибка/Удаление | Смягчённый красный |

> Дизайнер волен предложить свой вариант палитры в рамках «тёплого/детского» настроения.

### Типографика

- **Заголовки:** Скруглённый sans-serif (направление: Nunito, Comfortaa, или подобный)
- **Основной текст:** Системный шрифт для максимальной читаемости
- **Кириллица обязательна** во всех шрифтах

### Компоненты

- Скруглённые углы: **12-16px** на карточках и кнопках
- Мягкие тени (spread 8-16px, opacity 5-10%)
- Иллюстрации для пустых состояний — стиль «рисунок от руки» или мягкий flat
- Иконки: outline-стиль, скруглённые линии (направление: Phosphor, Lucide)

## 7. Технические ограничения Telegram Mini App

**Критично для дизайнера** — Telegram Mini App ≠ обычное веб-приложение:

| Ограничение | Что учитывать в дизайне |
|---|---|
| **Ширина экрана** | Фиксирована ~100vw, мобильное разрешение (375-428px). Десктоп не нужен |
| **Header** | Нативный TG header (кнопка «Назад» + заголовок). Кастомный header не нужен |
| **MainButton** | Основная кнопка действия — нативная TG кнопка внизу экрана. Настраивается: текст, цвет, состояние (активна/неактивна/лоадер). Не рисовать свою кнопку для главного действия |
| **Safe area** | Учитывать отступы сверху (под TG header) и снизу (под MainButton + home indicator) |
| **Тема** | Обязательна поддержка светлой и тёмной темы Telegram. TG предоставляет CSS-переменные |
| **Жесты** | Свайп назад управляется Telegram, не приложением. Не проектировать жест-навигацию |
| **Шрифты** | Веб-шрифты грузятся медленно в Mini App. Предпочтительнее системные. Декоративные — только для заголовков |
| **Медиа** | Фото/видео загружаются через бэкенд (URL), не напрямую из TG. Учитывать задержку загрузки |
| **Размер bundle** | Чем легче — тем быстрее открытие. Минимум тяжёлых ассетов |

## 8. Данные для мокапов

Для реалистичных макетов использовать:

| Поле | Значение |
|---|---|
| Имя ребёнка | Вика |
| Возраст | 1 год 3 мес |
| Автор 1 | Серёжа |
| Автор 2 | Настя |
| Пример тегов | #первый-раз, #еда, #сон, #прогулка, #здоровье, #игра, #развитие, #настроение, #купание, #зубы |

**Примеры записей:**
1. «Вика сегодня впервые сама держала ложку! Каша была везде, но она так радовалась 😊» + 2 фото
2. «Гуляли в парке, Вика увидела собаку и начала смеяться» + 1 видео
3. «Спала сегодня 2 часа днём — рекорд!» (только текст)
4. Голосовое сообщение: «Вика сказала "мама"...» (мини-плеер)

## 9. Deliverables

| # | Что | Формат |
|---|---|---|
| 1 | Логотип | 2-3 варианта в Figma (+ PNG/SVG экспорт) |
| 2 | UI Kit | Figma: палитра (light + dark), типографика, компоненты с auto-layout |
| 3 | Экраны | Figma: 5 экранов × все состояния (см. раздел 5) |
| 4 | Иллюстрации | 3-4 иллюстрации для пустых состояний |
| 5 | Прототип | Кликабельный Figma-прототип основных сценариев |

### Основные сценарии для прототипа

1. **Просмотр ленты → открытие записи → просмотр фото в fullscreen**
2. **Создание новой записи с фото → сохранение → возврат в ленту**
3. **Редактирование записи → сохранение**
4. **Удаление записи с подтверждением**
5. **Просмотр саммари → переключение месяцев**
