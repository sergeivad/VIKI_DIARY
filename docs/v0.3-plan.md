# v0.3: `/summary` + Редактирование записей

## Context

v0.2 (транскрипция голосовых + авто-теги) завершена. Следующий шаг по дорожной карте — v0.3: конспект за месяц (`/summary`) и редактирование записей. Summary генерируется через GPT-4o. Редактирование в Telegram-боте — только текст, но сервисный слой проектируется с учётом будущего расширения (редактирование медиа в Mini App).

---

## Feature 1: `/summary` — Конспект за месяц

### Новые файлы

**`src/utils/month.ts`** — Хелперы для работы с месяцами:
- `getMonthDateRange(year, month)` → `{ dateFrom: Date, dateTo: Date }` (UTC date-only)
- `formatRuMonth(year, month)` → `"февраль 2026"`

**`src/services/summary.errors.ts`** — Ошибки по шаблону `diary.errors.ts`:
- `SummaryErrorCode.noEntries`, `SummaryErrorCode.generationFailed`
- `SummaryDomainError` class

**`src/services/summary.service.ts`** — Генерация конспекта:
- Constructor: `(openai: OpenAI, log: Logger)`
- `generateSummary(input: SummaryInput): Promise<string>`
  - Собирает все тексты из entries
  - Системный промпт: тёплый, родительский тон, на русском
  - Включает имя малыша, возраст, даты
  - Выделяет: вехи, первые разы, паттерны (сон/еда/прогулки), эмоциональные моменты
  - Модель: `gpt-4o`, max_tokens: 1500
  - При ошибке: throw `SummaryDomainError`

**`src/bot/keyboards/summary.ts`** — Клавиатура навигации:
- Callback формат: `summary:nav:{year}:{month}`
- `buildSummaryKeyboard(year, month)` — кнопки "Назад"/"Вперёд" по месяцам
- Не показывать "Вперёд" для текущего/будущего месяца

**`src/bot/handlers/summary.ts`** — Хендлер команды:
- Resolve user → baby → определить месяц (если <= 5-е число → предыдущий месяц)
- Fetch entries через `diaryService.getEntriesForDateRange()`
- Пустой месяц → "В {месяце} записей нет"
- `ctx.replyWithChatAction("typing")` перед LLM-вызовом
- Reply с конспектом + навигационная клавиатура

**`src/bot/handlers/summaryCallbacks.ts`** — Навигация по месяцам:
- Parse callback `summary:nav:{year}:{month}`
- Validate (не будущий месяц)
- Fetch entries + generate summary
- `editMessageText` с результатом + обновлённая клавиатура

### Изменения в существующих файлах

## Feature 2: Редактирование записей

### Новые файлы

**`src/bot/conversations/editEntry.ts`** — Conversation для редактирования:
- Payload: `{ entryId, actorId, currentText, sourceChatId, sourceMessageId }`
- Показывает текущий текст, просит ввести новый
- `conversation.waitFor("message:text")` с валидацией
- Вызывает `diaryService.updateEntryText()`
- Обновляет исходное сообщение с `buildEntryActionsKeyboard`
- Fire-and-forget: регенерация тегов после редактирования

### Изменения в существующих файлах

**`src/services/diary.service.ts`** — Новый метод:
```ts
updateEntryText(input: {
  entryId: string;
  actorId: string;
  newText: string;
}): Promise<DiaryEntryDTO>
```
- Access check через существующий `assertActorHasAccessToEntryTx` (private, строка 291)
- Находит все text-items, обновляет первый, удаляет остальные (консолидация)
- Если text-items нет — создаёт новый с `orderIndex: 0` (сдвигая существующие)
- Trim + проверка на пустоту

**`src/bot/keyboards/entryActions.ts`**:
- Добавить `buildEditCallbackData(entryId)` → `entry:edit:{entryId}`
- Обновить `buildEntryActionsKeyboard` — добавить кнопку "✏️ Редактировать" первой строкой

**`src/bot/handlers/callbacks.ts`**:
- Добавить `{ type: "edit"; entryId: string }` в `ParsedEntryCallback`
- Парсинг: `parts[1] === "edit" && parts.length === 3`
- Обработка: fetch entry → extract text → `ctx.conversation.enter("editEntryConversation", payload)`

**`src/bot/bot.ts`**:
- `bot.use(createConversation(editEntryConversation, "editEntryConversation"))` — рядом с другими conversations
